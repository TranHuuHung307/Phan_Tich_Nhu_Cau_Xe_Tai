# Freight Demand Data Analysis

This project analyzes weekly freight demand by route and fleet type. It covers
data validation, cleaning, visualization, and additive time-series decomposition:

`Observed = Trend + Seasonality + Residual`

## Project Structure

```text
data/
  raw/
    sales.csv
    Du_Bao_Nhu_Cau_Xe_Tai_Mua_Vu_5Nam.xlsx
  processed/
    sales_cleaned.csv
    sales_cleaned.csv.gz
notebooks/
  01_eda_analysis.ipynb
reports/
  final_report.pdf
  figures/
src/
  clean_data.py
  visualize.py
  utils.py
requirements.txt
README.md
```

The files in `data/raw/` are source data and must never be edited directly.
The cleaning pipeline only reads `sales.csv` and fails if it is missing.
`sales_cleaned.csv` is reproducibly generated by `src/clean_data.py`.
The same cleaned dataset is also exported as `sales_cleaned.csv.gz` for
compressed storage.
Verify raw-file integrity from the project root with:

```bash
sha256sum -c data/raw/SHA256SUMS
```

## Setup

```bash
# Run these commands from the project root.
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m src.clean_data
python -m src.visualize
python -m src.generate_report
jupyter notebook notebooks/01_eda_analysis.ipynb
```

On Windows, activate the environment with `.venv\Scripts\activate`.

## Cleaning Rules

- Normalize string values and compact numeric dtypes.
- Check missing values, duplicate rows, duplicate business keys, non-positive
  metrics, valid week/event domains, and `Year_Week` consistency.
- Fill missing metrics only within the same `Route + Fleet_Type` series.
- Assign ISO weeks to months using Thursday, keeping most of each week in one
  month and keeping `2025-W01` in January 2025.
- Flag anomaly candidates on residuals after removing route/fleet trend and
  week-of-year seasonality.
- Exclude only rows explicitly marked `Is_Peak_Event = 1` from anomaly flags;
  non-peak route/fleet rows in mixed weeks remain eligible for review.
- Retain anomaly candidates instead of automatically deleting or winsorizing
  them because they may represent real operational events.

## Analysis Summary

The raw dataset contains 1,040 rows with no missing values, duplicate keys,
non-positive metrics, or invalid week/event values. Demand has a clear upward
trend. Monthly comparisons and decomposition use average weekly demand to avoid
bias from months containing different numbers of weeks. Large residuals are
treated as investigation candidates, not automatically as data errors.

The required visualization set is available in both the notebook and
`src/visualize.py`: line chart, bar chart, pie chart, and scatter plot.

## Report

`reports/final_report.pdf` is generated from the processed data, decomposition
results, and all report figures:

```bash
python -m src.generate_report
```

The fully executed notebook is also exported to `reports/final_report.html`.
To regenerate the HTML:

```bash
jupyter nbconvert --to pdf notebooks/01_eda_analysis.ipynb \
  --output-dir reports --output final_report
```
